# О проекте Frilans

Личный учёт фриланса: проекты, задачи, таймер и отчёты в одном месте. Задуман как небольшое приложение без лишней инфраструктуры — один процесс, файл SQLite, настройки в INI.

## Стек

| Слой | Технологии |
|------|------------|
| Backend | Python 3.11+, FastAPI, Starlette |
| Данные | SQLAlchemy 2, SQLite |
| Шаблоны | Jinja2 |
| Сессии | signed cookie (`itsdangerous`) |
| Пароли | bcrypt через passlib |
| Почта | стандартный `smtplib`, письма текст + HTML |
| Фронт | немного CSS (темы light/dark/system), без SPA |

## Как устроено

- **`config.ini`** — имя приложения, debug, техработы, base URL, секрет сессии, БД, учётка админа, SMTP. Часть полей можно менять из админки; значения перечитываются без перезапуска (кроме смены `database_url`, там нужен рестарт процесса).

- **Почта**: подтверждение регистрации и сброс пароля идут **шестизначным кодом**. В базе хранится HMAC-SHA256 от кода с привязкой к `user_id`, срок жизни кода ограничен (минуты настраиваются константами в `app/main.py`).

- **Роли**: флаг `is_admin` у пользователя; админ видит `/admin`, может блокировать пользователей, крутить техработы и SMTP.

- **Техработы**: middleware отдаёт заглушку всем, кроме статики, входа/выхода, документов (`/guide`, `/privacy`, `/terms`), потока регистрации/кодов и админки.

## Структура каталогов (кратко)

```
app/
  main.py          — маршруты, middleware, почта, отчёты
  config.py        — загрузка настроек
  models.py        — ORM-модели
  database.py      — engine, сессии
  deps.py          — зависимости «текущий пользователь», админ
  auth_utils.py    — пароли, коды для почты
  mailer.py        — отправка письма
  formatting.py    — деньги, даты
  services/        — расчёт заработка и т.д.
templates/       — Jinja
static/          — CSS, JS
```

## Что уже сделано по сути

- Регистрация / вход / выход, блокировка пользователя.
- Подтверждение почты и сброс пароля **по коду из письма**.
- Проекты (почасовая и фиксированная оплата), задачи, подзадачи, теги.
- Учёт времени вручную и таймером; активный таймер один на пользователя.
- Главная с метриками, отчёты, экспорт CSV.
- Настройки профиля, темы, админ-панель, режим техработ, документы в футере («Как пользоваться», политика, правила).

## Запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Демо-данные: `python example.py` (подставляет тестовых пользователей и проекты).

## Заметки по деплою

За reverse proxy выставьте заголовки `Host` и при необходимости `X-Forwarded-Proto`, чтобы ссылки в письмах собирались правильно; поле **base URL** в конфиге — запасной вариант.

Старые файлы SQLite подхватывают колонку `verification_expires_at` при старте (см. `app/database.py`); тяжёлые миграции не подключались намеренно.
